Code Setup Details

  1. Open PHPStorm
  2. Navigate to prototypes
    • Insert this command in the terminal tab
      • cd prototypes then press Enter
    • This changes the directory to the prototypes folder
    • If you get an error with the above command it's probably because you are already within the correct directory
  3. Create a feature grouping (Create feature branch)
    • Insert & execute these commands in the terminal tab individually
      • git checkout master then press Enter
        • The command above switches to the default branch
      • git pull https://github.com/Learning-Fuze/{{ repoName }}.git then press Enter
        • The command above downloads the newly posted prototypes and saves them to your local machine
      • git checkout -b {{ branch }} then press Enter
        • Creating this feature grouping allows users to modify code without causing issues with the current functioning code
        • Any modifications to these files after branching can be undone
        • "{{ branch }}" is the branch name

Coding Instructions

After Coding Details

  1. Save files to Git - Add all modified & created files
    • Run below command in the terminal tab
      • git add . then press Enter
    • Saving files to Git registers the changed files in Git, allowing them to be grouped together by a message (commit)
  2. Package (group) all saved git files with a commit message
    • Run below command in the terminal tab
      • git commit -m "{{ branch }} - Eric Johnson" then press Enter
        • Eric Johnson should be replaced with your First & Last name
        • Grouping files with a message allows those changes to be seen by others as a specific version of the code
  3. Allow others to see your changes by uploading them to the Github website
    • Run below command in the terminal tab
      • git push origin {{ branch }} then press Enter
        • When running the code above, the Git application will send all commits (groupings of saved files) to the remote location (https://github.com/[your-user-name]/{{ repoName }} - indicated by origin), specifically to the feature branch/group {{ branch }}
  4. See if your feature fits with what everyone else was working on (Pull Request)

      Step 4.1 Open your prototypes repo in the browser

      • The link should follow this format https://github.com/[your-user-name]/prototypes

      Step 4.2 Click Pull requests tab to open the pull request list

      Step 4.3 Click New pull request button

      Step 4.4 Compare Changes

      During this step there are a hand full of highlighted things that need to be taken into consideration

      1. base fork: Learning-Fuze/{{ repoName }}
        1. For all prototypes this will always remain the same
      2. base: C7_username
        1. Here you will select the option that matches YOUR username
        2. If you can not locate an option that has your username, contact an instructor
      3. head fork: [your_user_name]/{{ repoName }}
        1. Here you will select the option that matches your username / {{ repoName }}
        2. If you can not locate an option that has your username / {{ repoName }}, contact an instructor
      4. compare: {{ branch }}
        1. Here you will locate the branch name (or feature name) you created at the beginning of the prototype
        2. This will usually be the same as the name of the prototype
      5. Github.com will let you know if the code can be merged automatically (without needing a manual merge)
        1. If the text says soething other than "able to merge", Contact an instructor
        2. Sometimes a manual merge is necessary, it just has a couple

      Step 4.5 Add details into the pull request

      1. Give the pull request a title, by default it's the name of the last commit (package) message that was saved
        1. If you set the commit message above, you should get {{ branch }} - [your first & last name] already pre-populated in the Title Field
        2. If there is a different title pre-populated, change the title to match the correct format:
          1. {{ branch }} - [your first & last name]
      2. Don't assign a user. (An Instructor will select who this get assigned too)
        1. Further into the cohort we will have students learn to review pull requests
      3. Click on the Create pull request button
  5. Collaborate on feedback
    • Within the Github.com UI another developer or in this case an instructor will review the Pull Request you made - Is your feature working. Collaboration happens through Github UI & the notifications are sent through email when when comments are added to pull requests.
      • Students should make modifications to their code based on the instructors feedback
      • Instuctors will be giving an initial assessment based on the pull request. Any additional collaboration after the instructor's initial feedback will be on a requested basis only.
  6. Pull request Closed
    • In the LearningFuze workflow the pull requests are merged and closed to indicate assessment has been given to the student and recorded
    • The pull request will be merged into your branch on the LearningFuze {{ repoName }} repo
